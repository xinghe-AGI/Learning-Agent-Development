# 大语言模型 LLM

参考资料：

- [Attention Is All You Need](https://arxiv.org/abs/1706.03762)

## 什么是 LLM？

**LLM（Large Language Model，大语言模型）是一类使用大规模数据和参数训练、用于理解与生成语言及其他序列信息的深度神经网络。** 目前主流生成式 LLM 大多基于 Transformer，但模型的训练数据、架构细节和后训练方法并不完全相同。

## 训练与推理是两个阶段

| 阶段 | 模型在做什么 | 是否更新参数 |
| --- | --- | --- |
| 训练 | 根据训练目标计算误差，并通过优化算法调整参数 | 是 |
| 推理 | 使用已经训练好的参数，根据输入计算输出 | 否，常规调用不会修改模型参数 |

许多生成式 LLM 在预训练阶段使用[下一个 Token 预测](<下一个词预测训练目标.md>)。为了成为更好用的对话助手，模型通常还会经历指令微调、偏好优化、安全训练等后训练过程。

## Prompt 如何变成输出？

1. **输入上下文**：系统指令、用户问题、历史消息、工具结果等组成 Prompt。
2. **Tokenization**：Tokenizer 把文本转换为 [Token](<Token.md>) ID。
3. **向量表示**：Token ID 转换为模型可以计算的向量，并加入位置信息。
4. **上下文计算**：[Transformer](<Transformer 架构.md>) 通过多层注意力与前馈网络处理上下文。
5. **预测分布**：模型为下一个 Token 计算概率或分数。
6. **选择 Token**：解码策略根据概率分布选出一个 Token。
7. **追加并重复**：新 Token 加入上下文，继续预测，直到停止条件满足。

![LLM 自回归生成流程](<pictures/foundation-03-llm-generation-loop.png>)

这就是[自回归生成](<自回归生成方式.md>)：**每一步只决定下一个 Token，完整回答由许多局部预测连续组成。**

## 采样参数影响什么？

| 参数或策略 | 主要作用 | 常见影响 |
| --- | --- | --- |
| Temperature | 调整概率分布的平坦程度 | 低温度更稳定，高温度更多样 |
| Top-p | 只在累计概率达到阈值的候选中采样 | 限制长尾低概率 Token |
| Top-k | 只保留概率最高的 k 个候选 | 控制候选范围 |
| Max tokens | 限制最多生成多少 Token | 控制长度、成本与延迟 |
| Stop sequence | 遇到指定序列时停止 | 便于结构化截断 |

不同模型或 API 对这些参数的支持和解释可能不同。**采样参数改变的是生成策略，不会为模型补充缺失知识。**

## LLM 为什么看起来会“理解”和“推理”？

为了在复杂文本中持续预测合适的后续 Token，模型需要学习语法、语义、事实关联、代码结构和常见推理模式。这些能力以分布式方式存在于参数和运行时上下文中。

但流畅输出不等于事实正确。LLM 仍可能：

- 根据不完整上下文生成看似合理的错误内容。
- 对时效性、私有或冷门事实缺少可靠信息。
- 在多步推理和精确计算中出现局部错误。
- 受到 Prompt、上下文顺序和采样策略影响。

## LLM、RAG、工具和 Agent

LLM 本身负责根据上下文生成输出；应用可以在它周围增加其他能力：

- **RAG**：先检索资料，再把相关证据放入上下文。
- **工具调用**：让应用根据模型输出调用 API、数据库或代码。
- **Agent/工作流**：围绕目标编排多次模型调用、工具执行与状态更新。
- **评估与监控**：记录输出质量、错误、延迟和成本。
