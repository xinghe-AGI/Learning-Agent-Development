# 强化学习 Reinforcement Learning

## 什么是强化学习？

**强化学习让智能体通过与环境交互，根据奖励信号学习策略，目标是提高长期累积回报。** 它关注的不是一次预测是否正确，而是一连串动作最终带来的结果。

## 核心要素

| 要素 | 含义 |
| --- | --- |
| 智能体（Agent） | 做出选择的学习者 |
| 环境（Environment） | 智能体行动并获得反馈的外部世界 |
| 状态（State） | 当前可用于决策的信息 |
| 动作（Action） | 智能体可选择的行为 |
| 奖励（Reward） | 环境对动作结果给出的反馈信号 |
| 策略（Policy） | 根据状态选择动作的方法 |
| 价值（Value） | 对未来累积回报的估计 |

## 基本交互循环

```text
观察状态 -> 选择动作 -> 环境变化 -> 获得奖励和新状态 -> 更新策略 -> 重复
```

强化学习的难点在于**延迟反馈**和**探索与利用的平衡**。一个动作可能当下没有收益，却为后续成功创造条件；智能体既要尝试新策略，也要利用已经学到的有效策略。

## 常见方法

- **Q-Learning、SARSA**：学习状态与动作的价值。
- **DQN**：用神经网络近似 Q 值。
- **Policy Gradient**：直接优化策略参数。
- **Actor-Critic**：一部分模型负责选择动作，另一部分估计价值并提供学习信号。
- **PPO**：通过限制更新幅度提高策略优化的稳定性。

## 与 LLM 的关系

LLM 的大规模预训练通常不是强化学习，而是自监督的下一个 Token 预测。强化学习更多出现在后训练或 Agent 决策中，例如根据人类或模型反馈调整行为偏好。

**不能把“LLM 会生成答案”直接理解为“LLM 一直在做强化学习”。**

## 适用场景

- 决策会连续影响后续状态。
- 没有每一步的标准答案，但可以定义结果好坏。
- 环境可以模拟，或者真实试错成本可控。
- 目标需要在长期收益和短期收益之间取舍。

## 局限性

- 训练样本效率可能较低。
- 奖励设计不当会让智能体钻规则漏洞。
- 真实环境中的探索可能带来安全风险。
- 策略在新环境中可能失效，需要持续评估。
