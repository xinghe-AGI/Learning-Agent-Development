# 下一个 Token 预测训练目标

## 核心概念

**许多生成式 LLM 在预训练时，会根据前面的 Token 预测真实的下一个 Token，并用预测误差更新模型参数。** 这是一种自监督学习：训练目标可以直接从原始文本向右移动得到，不需要人为给每句话标注类别。

## 训练样本怎样构造？

假设文本被切分为：

```text
[今天, 天气, 很, 好, 。]
```

模型可以从同一序列得到多组训练位置：

| 上文 | 训练目标 |
| --- | --- |
| `[今天]` | `天气` |
| `[今天, 天气]` | `很` |
| `[今天, 天气, 很]` | `好` |
| `[今天, 天气, 很, 好]` | `。` |

实际训练会并行计算序列中多个位置的损失，不需要像推理那样真的逐步等待每个 Token 生成。

## 模型如何更新参数？

1. 模型为词表中的候选 Token 计算分数。
2. Softmax 将分数转换为概率分布。
3. 交叉熵损失衡量真实 Token 的预测概率是否足够高。
4. 反向传播计算梯度。
5. 优化器更新参数，使模型下次更倾向于真实后续。

重复大量样本后，模型为了降低预测误差，会学习语言结构、文本模式、事实关联、代码规律和部分推理模式。

## 为什么简单目标能形成复杂能力？

准确预测后续 Token 经常需要同时利用：

- 语法和词语搭配。
- 当前文档的主题与上下文。
- 文本中反复出现的事实关联。
- 数学、代码和逻辑模式。
- 指代、语气与文体结构。

但“学会预测”不等于形成可靠的事实数据库。模型优化的是概率目标，因此仍可能生成统计上合理但事实错误的内容。

## 需要补充的边界

- 下一个 Token 预测不是所有模型的唯一训练目标。
- 聊天模型通常还会经历指令微调、偏好优化和安全训练。
- 多模态模型可能同时使用图像、音频或对比学习目标。
- 训练目标解释了基础能力来源，但不能单独解释最终产品行为。

## 与自回归生成的关系

**训练时用真实文本作为上文，推理时用模型自己生成的 Token 继续向后。** 这会带来训练与生成上下文之间的差异，也解释了为什么生成早期的错误可能影响后续内容。
