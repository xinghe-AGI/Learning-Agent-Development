# Transformer 架构

参考资料：

- [Attention Is All You Need](https://arxiv.org/abs/1706.03762)

## Transformer 是什么？

**Transformer 是一种以注意力机制为核心的神经网络架构，最初用于序列到序列任务，后来成为主流 LLM 的基础架构。** 它不依赖 RNN 的逐步递归来传递序列信息，因此训练时更容易并行处理多个位置。

## 核心组成

| 组成 | 作用 |
| --- | --- |
| Token Embedding | 把 Token ID 映射为连续向量 |
| 位置信息 | 告诉模型 Token 在序列中的位置或相对距离 |
| 多头自注意力 | 让每个位置从其他位置聚合相关信息 |
| 前馈网络（FFN） | 对每个位置的表示进行非线性变换 |
| 残差连接 | 保留原始信息并帮助深层网络训练 |
| 归一化 | 稳定数值分布和训练过程 |

这些模块会重复堆叠多层。模型在每一层更新 Token 的表示，使其逐渐包含更丰富的上下文信息。

## 编码器与解码器

原始 Transformer 同时包含编码器和解码器。后续模型常见三种形态：

| 形态 | 关注方式 | 常见任务倾向 |
| --- | --- | --- |
| Encoder-only | 可以同时利用左右上下文 | 表示、分类、抽取 |
| Decoder-only | 使用因果遮罩，只看当前位置之前的 Token | 自回归生成、对话、代码生成 |
| Encoder-Decoder | 编码输入，再由解码器生成输出 | 翻译、摘要、条件生成 |

大多数聊天式生成模型使用 Decoder-only 结构，但具体架构会加入不同的位置编码、归一化、注意力优化或稀疏专家等设计。

## 为什么适合大语言模型？

- **训练可并行**：同一序列中的多个位置可以同时参与计算。
- **长距离依赖**：注意力可以直接连接相距较远的位置。
- **结构可扩展**：层数、隐藏维度、注意力头和专家模块可以扩展。
- **任务通用**：同一基本架构可用于文本、图像、语音和多模态任务。

## 主要限制

- 标准全注意力对序列长度的计算和显存开销较大。
- 能关注长上下文，不代表一定能正确使用所有长上下文信息。
- Transformer 只是架构，模型能力还取决于数据、训练目标、后训练和推理系统。
