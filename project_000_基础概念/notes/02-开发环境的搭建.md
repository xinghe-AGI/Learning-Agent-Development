# AI 应用开发环境搭建

参考资料：

- [LangChain installation](https://docs.langchain.com/oss/python/langchain/install)
- [Anaconda documentation](https://docs.anaconda.com/)
- [PyCharm documentation](https://www.jetbrains.com/help/pycharm/)
- [Ollama documentation](https://docs.ollama.com/)

这篇笔记以 Windows + Python 为例，目标是建立一个**依赖隔离、凭据不入库、可以复现**的基础开发环境。

## 1. 准备工具

| 工具 | 作用 | 说明 |
| --- | --- | --- |
| Python / Conda | 运行环境与依赖隔离 | LangChain 当前要求 Python 3.10+ |
| PyCharm 或 VS Code | 编辑、调试和运行代码 | 任选熟悉的 IDE |
| Git | 版本管理 | 不要提交密钥和本地环境目录 |
| 模型服务或 Ollama | 提供模型推理能力 | 云端 API 与本地模型二选一或组合 |

下载入口：

- [Anaconda](https://www.anaconda.com/download)
- [PyCharm](https://www.jetbrains.com/pycharm/download/)
- [Git](https://git-scm.com/downloads)
- [Ollama](https://ollama.com/download)

## 2. 创建独立 Python 环境

建议一个项目使用一个环境，环境名与项目名保持一致，便于识别。

```powershell
conda create -n agent-learning python=3.12
conda activate agent-learning
python --version
python -m pip install --upgrade pip
```

更多命令见[conda 常用命令](<conda命令.md>)。

## 3. 创建项目并绑定解释器

在 IDE 中创建项目后，把 Python Interpreter 指向刚才创建的 Conda 环境。验证终端和 IDE 使用的是同一个解释器：

```powershell
python -c "import sys; print(sys.executable)"
```

如果路径不是预期环境，先修正解释器设置，再安装依赖。

## 4. 选择模型接入方式

### 云端模型 API

优点是模型能力强、无需本地部署；需要关注费用、网络、数据合规和服务稳定性。优先使用模型厂商官方 SDK 或可信的企业网关。

### OpenAI 兼容接口

一些服务提供 OpenAI 风格的 API，但“接口兼容”不代表能力、参数和安全策略完全一致。需要确认：

- 服务主体与隐私条款。
- 数据是否被记录或用于训练。
- 模型名称、上下文限制与计费方式。
- 是否支持工具调用、结构化输出和流式响应。

### 本地模型

Ollama 等工具适合本地学习和隐私敏感实验，但效果、速度和可用上下文取决于模型与硬件。

## 5. 安装项目依赖

先安装真正需要的最小依赖：

```powershell
python -m pip install -U langchain langgraph
python -m pip install -U langchain-openai
```

如果项目需要固定版本，先验证可运行组合，再写入依赖文件：

```powershell
python -m pip freeze > requirements.txt
```

学习笔记中可以记录验证过的版本，但不要把某个时间点的版本写成永久推荐。

## 6. 安全管理 API 凭据

不要把 API Key 写进 Python 文件、Markdown、截图或 Git 仓库。可以使用系统环境变量或本地 `.env` 文件，并把 `.env` 加入 `.gitignore`。

```powershell
$env:OPENAI_API_KEY="仅在当前终端临时设置"
```

仓库中只保留不含真实值的示例文件：

```text
# .env.example
OPENAI_API_KEY=
MODEL_NAME=
```

## 7. 最小验收

完成环境后检查：

- `python --version` 符合依赖要求。
- `python -c "import sys; print(sys.executable)"` 指向正确环境。
- `python -m pip check` 没有依赖冲突。
- API 凭据没有出现在仓库文件中。
- 能运行一次最小模型调用，并对超时或认证失败给出清晰错误。
