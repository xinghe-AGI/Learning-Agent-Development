# 自回归生成方式 Autoregressive Generation

## 什么是自回归生成？

**自回归生成是一种逐步生成序列的方式：每次生成一个新 Token，再把它加入已有上下文，用来预测下一个 Token。**

```text
P(x1, x2, ..., xn)
= P(x1) * P(x2|x1) * P(x3|x1,x2) * ... * P(xn|x1,...,xn-1)
```

## LLM 的生成循环

1. Prompt 被切分为 Token 并输入模型。
2. 模型计算下一个 Token 的概率分布。
3. 解码策略选择一个 Token。
4. 新 Token 追加到上下文。
5. 模型再次计算下一个 Token。
6. 遇到停止 Token、停止序列或长度上限时结束。

```text
输入：“今天天气”
第 1 步：“今天天气” -> “很”
第 2 步：“今天天气很” -> “好”
第 3 步：“今天天气很好” -> “。”
```

真实 Token 切分不一定与汉字逐字对应，这里只展示生成逻辑。

## 与下一个 Token 预测的关系

[下一个 Token 预测](<下一个词预测训练目标.md>)描述训练目标，自回归描述生成过程。二者紧密相关，但不是同一个概念：

| 概念 | 发生阶段 | 重点 |
| --- | --- | --- |
| 下一个 Token 预测 | 主要用于预训练 | 根据真实上文学习预测真实后续，并更新参数 |
| 自回归生成 | 推理 | 根据模型自己的已生成内容继续预测，不更新参数 |

## 这种方式带来的影响

- **前面会影响后面**：早期错误可能进入上下文并持续放大。
- **生成难以完全并行**：下一个 Token 要等待前一个 Token 确定。
- **中间文本可以帮助后续计算**：分步输出为后续 Token 提供更多上下文，但不能保证推理一定正确。
- **停止条件很重要**：长度上限和停止序列会直接影响输出完整性。

## 常见误区

- 自回归不等于模型像人一样先想好完整答案。
- 每一步选择最高概率 Token 不一定得到整体最优文本。
- 温度变低只能减少随机性，不能自动消除事实错误。
- 思维链效果不应只归因于自回归机制，还受训练数据、模型能力、任务结构和提示方式影响。
