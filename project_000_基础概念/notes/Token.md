# Token（词元）

## Token 是什么？

**Token 是 Tokenizer 按照自身词表和切分规则，把输入转换后得到的离散单位。** 它可能对应一个字、一个词、一个子词、标点、空格的一部分，也可能对应其他模态中的离散表示。

Token 不是天然存在于文本里的固定边界。同一段文字交给不同模型的 Tokenizer，切分结果和 Token 数量都可能不同。

## 为什么不直接按字或单词处理？

| 切分方式 | 优点 | 问题 |
| --- | --- | --- |
| 按字符 | 词表较小，几乎不会遇到未知词 | 序列很长，语义单位被拆得太碎 |
| 按完整单词 | 单个 Token 语义较完整 | 词表巨大，生僻词和新词难处理 |
| 按子词 | 在词表大小和序列长度之间折中 | 切分结果不一定符合人的词语直觉 |

许多语言模型使用 BPE、WordPiece、Unigram 或其变体进行子词切分。

## Token 如何进入模型？

```text
文本 -> Tokenizer -> Token ID -> Embedding 向量 -> Transformer
```

Tokenizer 负责把文本映射到 Token ID；Embedding 层再把离散 ID 转换为连续向量。**模型计算的不是文字本身，而是这些向量表示。**

## Token 数量为什么重要？

- **上下文窗口**通常用 Token 数衡量，输入与输出会共同占用窗口。
- **成本**常按输入和输出 Token 计量，具体规则取决于模型服务。
- **延迟**与处理、生成的 Token 数量有关。
- **截断**可能让模型看不到较早或较后的重要信息。
- **结构化输出**中的冗余字段也会占用上下文和生成预算。

不要用“一个汉字一定等于一个 Token”或“一个英文单词一定等于一个 Token”来估算。需要精确数量时，应使用目标模型对应的 Tokenizer。
