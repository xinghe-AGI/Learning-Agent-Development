# Prompt（提示词）

## Prompt 是什么？

**Prompt 是模型在一次推理中获得的输入上下文，用来说明任务、提供信息并约束输出。** 在聊天应用中，它不只包括用户最后一句话，还可能包括系统指令、历史消息、示例、检索结果和工具返回值。

对 LLM 来说，Prompt 会被转换为 Token，并成为后续[自回归生成](<自回归生成方式.md>)的条件上下文。

## Prompt 的常见组成

| 组成 | 作用 | 需要回答的问题 |
| --- | --- | --- |
| 任务指令 | 说明要完成什么 | 模型最终要做哪件事？ |
| 上下文 | 提供背景、资料和约束 | 哪些信息会影响判断？ |
| 输入数据 | 明确本次要处理的对象 | 哪段文本、数据或问题是输入？ |
| 示例 | 展示期望模式 | 什么样的输入应该得到什么样的输出？ |
| 输出要求 | 约束结构、语言和长度 | 结果要怎样被人或程序使用？ |
| 边界与禁止项 | 说明不能做什么 | 哪些风险、假设或格式必须避免？ |

这些组成是检查清单，不是必须照抄的固定模板。**Prompt 的关键不是越长越好，而是让任务所需的信息足够明确。**

## 一个最小示例

```text
任务：把下面的会议记录整理成行动项。

输入：
<meeting_notes>
{{会议记录}}
</meeting_notes>

输出要求：
- 使用 Markdown 表格。
- 列为：行动项、负责人、截止时间、待确认信息。
- 原文没有的信息写“待确认”，不要猜测。
```

这个 Prompt 没有强行指定角色，因为任务本身已经足够明确。角色、示例和复杂模板只在确实能减少歧义时添加。

## 如何判断 Prompt 问题？

- **输出偏题**：检查任务目标和输入边界是否清楚。
- **格式不稳定**：给出明确字段、JSON Schema 或示例。
- **事实不可靠**：补充可信资料，或使用 RAG/搜索，而不是继续堆形容词。
- **复杂任务质量低**：拆成多步工作流，每步单独评估。
- **同一错误反复出现**：建立测试样本，比较 Prompt 版本，而不是凭感觉修改。
