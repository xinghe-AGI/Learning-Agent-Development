# 监督学习 Supervised Learning

## 什么是监督学习？

**监督学习使用带标签的样本，学习从输入到目标输出的映射。** 标签相当于训练阶段的参照答案，模型通过比较预测与标签之间的差距来更新参数。

## 两类常见任务

| 类型 | 输出 | 示例 |
| --- | --- | --- |
| 分类（Classification） | 离散类别或类别概率 | 垃圾邮件识别、图片类别、客户是否流失 |
| 回归（Regression） | 连续数值 | 房价、需求量、设备剩余寿命 |

## 训练与预测

```text
训练：输入 + 标签 -> 计算预测 -> 计算损失 -> 更新模型参数
预测：新输入 + 已训练模型 -> 输出预测
```

监督学习能否泛化，关键不只在模型，还取决于训练数据是否覆盖真实场景，以及测试集是否与训练过程保持独立。

## 一个最小示例

```python
from sklearn.linear_model import LogisticRegression

# 两个特征：近 30 天使用次数、是否开通自动续费
X = [[2, 0], [5, 0], [18, 1], [24, 1], [30, 1]]
y = [0, 0, 1, 1, 1]  # 0=未续费，1=续费

model = LogisticRegression()
model.fit(X, y)

prediction = model.predict([[12, 1]])
print(prediction)
```

这个示例只用于理解流程。真实项目还需要更多样本、特征检查、数据划分、指标选择和偏差分析。

## 常见算法

- **线性与逻辑回归**：结构简单，适合作为基线。
- **决策树与集成方法**：适合表格数据，能表达非线性关系。
- **支持向量机**：适合中小规模、边界清楚的任务。
- **神经网络**：适合图像、语音、文本等复杂高维数据。

## 常见风险

- **标签质量差**：模型会学习标注错误与历史偏见。
- **数据泄漏**：训练时看到了真实部署阶段不应获得的信息。
- **类别不平衡**：准确率高，但少数类可能几乎全部判断错误。
- **分布漂移**：上线后的数据与训练数据不再相似。
- **指标选错**：模型分数提高，但业务结果没有改善。
