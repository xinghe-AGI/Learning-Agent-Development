# 自注意力机制 Self-Attention

参考资料：

- [Attention Is All You Need](https://arxiv.org/abs/1706.03762)

## 什么是自注意力？

**自注意力让序列中的每个位置，根据当前任务动态聚合同一序列中其他位置的信息。** “自”表示 Query、Key、Value 都来自同一组输入表示。

例如处理“它把球递给了小明，因为他站得更近”时，不同位置需要关注不同词语，才能形成与上下文相关的表示。注意力不是直接输出“谁指代谁”，而是为后续层提供加权后的信息。

## Query、Key、Value 的直觉

| 名称 | 直觉 | 作用 |
| --- | --- | --- |
| Query（Q） | 当前这个位置想找什么 | 与所有 Key 计算匹配程度 |
| Key（K） | 每个位置可以被怎样匹配 | 决定自己与 Query 的相关性 |
| Value（V） | 被关注后实际提供什么信息 | 按注意力权重加权汇总 |

![Query、Key、Value 与上下文表示](<pictures/foundation-04-attention-qkv.png>)

简化过程：

```text
输入表示
  -> 分别线性变换为 Q、K、V
  -> Q 与 K 计算相似度
  -> 缩放并经过 Softmax 得到注意力权重
  -> 用权重对 V 加权求和
  -> 得到融合上下文的新表示
```

常见公式：

```text
Attention(Q, K, V) = softmax(QK^T / sqrt(d_k))V
```

## 多头注意力为什么有用？

多头注意力并行使用多组 Q、K、V 投影。不同头可以学习不同类型的关系，例如局部搭配、语法依赖、指代线索或长距离关联，然后把结果合并。

**“多头”不保证每个头都对应一个人类可命名的规则，它只是提供多个表示子空间。**

## 因果遮罩

生成式 Decoder-only 模型会使用因果遮罩，使当前位置只能关注自己及之前的 Token，不能偷看未来 Token。这让训练方式与自回归生成目标保持一致。

## 局限性

- 标准注意力的计算量随序列长度快速增长。
- 注意力权重不等于完整的因果解释。
- 长上下文中的信息仍可能被忽略、稀释或错误关联。
- 自注意力只是 Transformer 的一个模块，不能单独解释模型全部能力。
