# 项目对比分析：从基础学习到 Prompt 模板功能

## 1. 功能差异对比

| 功能 | 01_Study_Quickstart | 02_Study_PromptTemplate |
|------|---------------------|-------------------------|
| 提示词管理 | 硬编码在代码中 | 外部文件管理 |
| 提示词模板 | 无 | 支持变量占位符 |
| 多角色提示 | 无 | 支持 system/human 角色 |
| 输入处理 | 固定消息 | 支持 Prompt 变量动态传参 |
| 提示词格式 | 固定文本 | 支持动态生成 |

## 2. 技术实现细节变化

### 2.1 新增依赖
- **新增导入**：从 `langchain_core.prompts` 导入 `PromptTemplate` 和 `ChatPromptTemplate`
- **模板文件**：新增 `prompt` 文件夹，包含系统提示和人类提示模板

### 2.2 配置管理
- **新增配置项**：在 `Config` 类中添加 `SYSTEM_PROMPT_TMPL` 和 `HUMAN_PROMPT_TMPL` 配置
- **模板路径**：配置模板文件的相对路径

### 2.3 提示词处理
- **从文件加载**：使用 `PromptTemplate.from_file()` 从外部文件加载模板
- **模板渲染**：使用 `ChatPromptTemplate.format_messages()` 动态渲染模板
- **多角色支持**：使用 `ChatPromptTemplate.from_messages()` 支持多种角色消息

### 3.2 02_Study_PromptTemplate/agent.py
```python
# 从外部文件加载提示词模板
system_prompt = PromptTemplate.from_file(
    template_file=Config.SYSTEM_PROMPT_TMPL,
    encoding="utf-8"
).template

human_prompt = PromptTemplate.from_file(
    template_file=Config.HUMAN_PROMPT_TMPL,
    encoding="utf-8"
).template

# 构建聊天提示模板
chat_prompt = ChatPromptTemplate.from_messages([
    ("system", system_prompt),
    ("human", human_prompt)
])

# 使用加载的系统提示词
agent = create_agent(
    model=llm_chat,
    system_prompt=system_prompt,
    # 其他参数...
)

# 动态渲染用户提示
messages = chat_prompt.format_messages(question=raw_question, name=name)
human_msg = messages[-1]

# 传递渲染后的用户消息
response = agent.invoke(
    {"messages": [{"role": "user", "content": human_msg.content}]},
    # 其他参数...
)
```

### 3.3 主要修改点
1. **提示词管理**：从硬编码改为外部文件管理
2. **模板渲染**：新增模板加载和渲染逻辑
3. **多角色支持**：使用 `ChatPromptTemplate` 支持多角色消息
4. **动态传参**：支持在运行时传入用户姓名和问题
5. **配置扩展**：新增模板文件路径配置

## 4. 新增 prompt 文件夹分析

### 4.1 组织结构
```
02_Study_PromptTemplate/
├── prompt/
│   ├── system_prompt_tmpl.md    # 系统提示模板
│   └── human_prompt_tmpl.md      # 人类提示模板
├── agent.py                      # 主入口文件
└── utils/
    └── config.py                 # 配置文件
```

### 4.2 文件类型及作用
- **system_prompt_tmpl.md**：系统提示模板，定义 Agent 的角色和行为规则
- **human_prompt_tmpl.md**：人类提示模板，定义用户输入的格式和要求

### 4.3 实现方式
1. **模板文件**：使用 Markdown 格式存储提示词模板
2. **变量占位符**：使用 `{variable}` 格式定义变量占位符
3. **加载机制**：使用 `PromptTemplate.from_file()` 加载模板文件
4. **渲染过程**：使用 `format_messages()` 方法替换变量占位符
5. **多角色组合**：使用 `ChatPromptTemplate.from_messages()` 组合不同角色的提示

### 4.4 模板内容示例
- **系统提示模板**：
  ```markdown
  # 角色：你是一名擅长讲冷笑话的专业天气预报员。

  ## 你可以使用两个工具：

  - get_weather_for_location：用于获取某个具体地点的天气
  - get_user_location：用于获取用户当前所在位置

  ## 规则：

  如果用户向你询问天气，一定要先确认地点。
  如果从问题中可以判断出用户指的是“我所在的地方”的天气，就使用 get_user_location 工具来获取用户的位置。
  ```

- **人类提示模板**：
  ```markdown
  请用搞笑的语气,结合冷笑话,回答用户关于天气的问题。用户的名字是：{name}。用户原始问题是: {question}
  ```

## 5. 架构调整与技术优势

### 5.1 架构调整
1. **分离关注点**：将提示词从代码中分离到外部文件
