# CoT 和自我一致性的区别

## 它们的核心关系

**CoT 负责生成一条可展开的推理链，自我一致性负责生成多条相互独立的推理链，再根据最终答案的一致程度进行聚合。**

两者经常组合使用，但不在同一个层级：

```text
CoT：问题 -> 一条推理链 -> 一个答案

Self-Consistency：
问题 -> 推理链 A -> 答案 X
     -> 推理链 B -> 答案 X
     -> 推理链 C -> 答案 Y
     -> 多数投票 -> 答案 X
```

## 它们的主要区别

| 对比维度 | CoT | 自我一致性 |
| --- | --- | --- |
| 核心动作 | 展开一次分步推理 | 多次采样推理链并聚合答案 |
| 主要目标 | 减少跳步，让中间过程更完整 | 降低单条推理路径的偶然错误 |
| 推理数量 | 通常一条 | 多条相互独立的推理链 |
| 输出重点 | 推理过程和最终答案 | 答案分布、票数和聚合结果 |
| 主要成本 | 增加单次输出 token | 增加模型调用次数和总 token |
| 适合问题 | 需要多步推导的问题 | 有明确答案但单次推理不稳定的问题 |
| 主要风险 | 一开始走错后可能一路错下去 | 多数路径可能共同偏向错误答案 |

**最关键的区别是：CoT 改善一条推理链，自我一致性比较多条推理链。**

## 什么时候先用 CoT？

当任务路线基本明确，只是模型容易省略步骤、漏掉条件或直接猜答案时，先使用 CoT。

CoT 更适合：

- 数学计算、逻辑推理和多条件判断。
- 希望检查模型在哪一步出错。
- 一次调用就有机会稳定完成的任务。
- 还没有证据表明多次答案会明显波动的任务。

如果一次 CoT 已经稳定正确，就没有必要为了“更高级”再增加多次采样。

## 什么时候升级为自我一致性？

**当模型已经会分步推理，但同一问题多次运行会得出不同答案时，再考虑自我一致性。**

常见触发信号包括：

- 同一个问题重复提问时，答案频繁变化。
- 不同推理路线都看似合理，难以只信任其中一条。
- 最终答案可以被标准化提取和投票。
- 错误成本高于多次调用带来的费用和延迟。

自我一致性更适合答案可以聚合的问题，例如数值、选项、标签或短结论；对开放写作和多种答案都合理的问题，多数投票的意义较弱。

## 自我一致性不一定总比 CoT 好

- **成本更高**：采样 5 次通常意味着接近 5 倍的推理开销。
- **投票不等于验证**：多数答案相同，也可能只是模型稳定地犯同一个错误。
- **答案必须可归一化**：`A`、`选项 A` 和一段自然语言解释需要先映射到同一答案。
- **简单任务收益有限**：事实问答和固定转换通常不需要多次推理。

高风险任务仍应使用计算器、单元测试、检索证据或人工审核进行外部验证。

## 一个实用决策顺序

1. 先用普通提示建立基线。
2. 推理容易跳步时加入 CoT。
3. 重复运行几次，观察最终答案是否稳定。
4. 单次 CoT 不稳定且答案可聚合时，加入自我一致性。
5. 多数投票仍不可靠时，增加外部验证，而不是继续无限采样。

**一句话判断：需要把一次推理做完整，用 CoT；需要降低单条推理的偶然性，用自我一致性。**

## 容易混淆的点

- 自我一致性通常以 CoT 为基础，但不是另一种更长的 CoT。
- 多条推理链应该尽量独立，不能让后一次直接照抄前一次答案。
- 温度和采样策略会影响路径多样性；完全相同的输出无法形成有效投票。
- 最终答案一致不代表每条中间推理都正确。
