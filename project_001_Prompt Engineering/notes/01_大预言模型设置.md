## 核心概念

### Temperature

`temperature` 控制输出的随机性。

值越低，模型越倾向选择概率最高、最稳妥的 token，回答会更确定、更保守。适合事实问答、摘要、分类、信息抽取这类希望稳定复现的任务。

值越高，模型会更愿意选择不那么显然的 token，回答会更发散、更有创造性。适合头脑风暴、改写、故事、广告文案、创意命名。

我的理解：temperature 像“冒险程度”。不是让模型更聪明，而是让它更敢选不那么常规的表达。

### Top_p

`top_p` 也控制随机性，但方式不同。它会限制模型只在累计概率达到某个阈值的一组候选 token 里选择。

`top_p` 低时，候选范围小，输出更确定；`top_p` 高时，候选范围大，输出更多样。

一般不要同时大幅调整 `temperature` 和 `top_p`。先改一个参数，更容易判断变化来自哪里。

### Max Length

`max length` 控制模型最多生成多少 token。

它的作用不是提升质量，而是约束长度、避免回答跑太远，也能控制成本。对于列表、摘要、结构化输出，设置最大长度很有用。

### Stop Sequences

`stop sequence` 是停止生成的字符串。模型一旦生成到这个字符串，就会停止继续输出。

它适合用来控制格式边界，比如让模型只生成到某个分隔符之前，或者限制列表不要无限延伸。

### Frequency Penalty

`frequency penalty` 会根据某个 token 已经出现的次数来惩罚它。出现越多，后面再出现的概率越低。

它主要用来减少“同一个词一直重复”的问题。

### Presence Penalty

`presence penalty` 也惩罚重复 token，但它只关心“出现过没有”，不太关心出现了几次。

如果想让模型多探索新话题、新表达，可以适当提高它；如果想让模型紧扣当前主题，不要太发散，就降低它。

`frequency penalty` 和 `presence penalty` 也不建议同时大幅调整，否则很难判断到底是哪一个造成输出变化。

## 参数怎么选

| 任务 | 推荐倾向 | 理由 |
|---|---|---|
| 事实问答 / QA | 低 temperature 或低 top_p | 更稳定、更少发散 |
| 摘要 / 信息抽取 | 低 temperature，限制 max length | 需要简洁、贴近原文 |
| 创意写作 / 头脑风暴 | 适当提高 temperature 或 top_p | 需要更多样的表达 |
| 固定格式输出 | 使用 max length 和 stop sequence | 控制长度和结构边界 |
| 输出经常重复 | 调整 frequency penalty 或 presence penalty | 降低重复词和重复表达 |

## 常见误区

- 误区一：把 temperature 调高等于回答质量更高。  
  实际上它只是让输出更随机，可能更有创意，也可能更不稳定。

- 误区二：同时调 temperature 和 top_p。  
  这样很难判断是哪一个参数影响了结果。更好的方式是一次只改一个变量。

- 误区三：只改 prompt，不看模型设置。  
  有时 prompt 没问题，是参数让输出太发散、太长或太重复。

- 误区四：以为所有模型版本表现都一样。  
  同样的 prompt 和参数，在不同模型或不同版本上可能有差异。
