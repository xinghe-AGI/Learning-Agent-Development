## 什么是零样本提示？

**零样本提示是一种不向模型提供示例，只通过任务指令让模型直接完成任务的提示方法**。这里的“零样本”指的是：prompt 里没有给模型展示“输入应该如何对应输出”的样例，模型需要依靠预训练中学到的语言知识、概念关联和指令理解能力来推断答案。

例如，直接给模型：

```text
将文本分类为中性、负面或正面。

文本：我认为这次假期还可以。
情感：
```

模型可能直接回答：

```text
中性
```

这就是典型的零样本提示：**没有给“正面 / 负面 / 中性”的示例，只给了任务说明和输入**。它依赖的是模型已经从大量数据中学到过类似的分类模式。

零样本提示通常是尝试新任务时的默认起点。因为它最简单，不需要准备样例，也不需要额外训练数据。**先看模型能不能直接做，如果效果不好，再考虑加入示例，进入少样本提示。**

## 零样本提示的工作原理

**零样本提示的核心在于利用大模型的预训练知识和指令理解能力。**

大模型在训练过程中看过大量自然语言文本、问答、分类、总结、解释、代码等模式，所以当用户给出一个清晰任务时，模型会尝试把当前 prompt 匹配到它熟悉的任务模式中，再生成最可能的输出。

具体来说：

- **任务指令**， 用户告诉模型要完成什么任务。例如“将文本分类为中性、负面或正面”“总结下面这段文字”“提取文本中的地点”。指令越清楚，模型越容易判断该走哪种生成模式。
- **上下文信息**， 有些零样本 prompt 虽然没有示例，但仍然可以给出类别定义、任务背景或判断标准。例如让模型判断问题优先级时，可以先说明“高、中、低”分别代表什么。这不算 few-shot，因为它没有给输入输出样例，而是在给模型判断依据。
- **输入数据**， 用户明确提供模型要处理的内容。例如待分类文本、待总结文章、待抽取信息的段落。输入最好和指令区分开，避免模型把任务说明和待处理文本混在一起。
- **输出指示符**， 有时 prompt 末尾会给出一个输出起点，例如“情感：”“类别：”“答案：”。它像一个接力棒，提示模型接下来应该生成什么类型的内容。

**零样本提示不是“模型凭空会做所有任务”，而是模型把当前任务映射到它已经学过的语言和任务模式里。** 如果任务太新、太专业、标准太隐含，零样本效果就会下降。

## 零样本提示的构造方式

零样本 prompt 可以很短，也可以包含比较完整的任务说明。**关键不是长度，而是有没有把任务说清楚。**

一个常见结构是：

```text
任务指令：你要模型做什么

判断标准或上下文：必要时说明类别、规则、背景

输入：明确标出待处理内容

输出指示：告诉模型输出什么
```

例如：

```text
请判断下面用户反馈的情感倾向，只能回答：正面、负面或中性。

用户反馈：
"""
我觉得这次更新还可以，但启动速度有点慢。
"""

情感：
```

如果任务标准比较简单，零样本提示可以只写指令和输入；**如果任务涉及分类边界、业务规则或专业判断，就应该补充上下文和判断标准。**

## 零样本提示的应用场景

零样本提示适合先快速验证模型是否能完成某类任务，尤其适合不方便准备样例、任务标准比较常见的场景。

- **文本分类**： 例如情感分类、优先级分类、意图识别、内容标签分类。只要类别定义清楚，模型经常可以直接给出可用结果。
- **信息抽取**： 例如从一段文本中提取人名、地点、时间、产品名、模型名称等。模型不需要示例，也能根据指令找到目标信息。
- **问答**： 对于常识性问题、开放领域问题，模型可以依靠预训练知识直接回答。
- **文本摘要**： 给出一段文本和“请总结”的指令，模型可以在没有摘要示例的情况下生成概要。
- **内容生成**： 例如生成文案、代码片段、邮件草稿、说明文、标题等。只要目标和约束清楚，零样本通常可以作为第一版。
- **对话任务**： 聊天模型本身经过指令调整和对话训练，很多日常问答、解释、改写、翻译任务都可以直接用零样本提示完成。

## 零样本提示的优势

- **简单易用**， 不需要准备输入输出示例，也不需要额外训练数据。对于刚开始探索一个任务时，零样本是成本最低的起点。
- **迭代快**， prompt 可以随时修改。发现输出不稳定时，可以补充指令、上下文、输出格式或判断标准。
- **适合数据稀缺场景**， 如果没有标注样本，或者样本还没整理好，零样本提示可以先帮助验证任务可行性。
- **泛化能力强**， 对于模型熟悉的通用任务，零样本提示经常能直接得到不错结果。尤其是经过指令调整和 RLHF 的模型，会更擅长理解用户自然语言指令。

## 零样本提示的局限性

- **性能不稳定**， 零样本提示很依赖模型自身能力和任务清晰度。任务越复杂、越专业、越依赖隐含规则，输出越容易波动。
- **缺少示例约束**， 模型不知道用户心中“好答案”的具体样子。对于格式转换、风格模仿、复杂分类标准，单靠指令可能不够。
- **依赖预训练覆盖范围**， 如果模型在训练中很少接触某个领域、语言、概念或任务模式，它的零样本表现会变差。
- **不适合复杂多步推理**， 对于数学推理、逻辑推理、规划决策这类任务，普通零样本提示可能直接跳到答案，容易出错。这时可以考虑[06_链式思考（CoT）提示](<06_链式思考（CoT）提示.md>)、[07_思维树 ToT](<07_思维树 ToT.md>)等推理增强方法。
- **容易受提示措辞影响**， 同一个任务换一种说法，模型输出可能明显变化。零样本提示虽然简单，但仍然需要调试。

## 零样本提示的使用经验

零样本提示的重点不是“什么都不写”，而是“不提供示例”。该给的任务说明、类别定义、输入边界、输出格式仍然要给。

写零样本 prompt 时，可以先问自己几个问题：

- 我有没有说清楚模型要做什么？
- 输入内容有没有明确标出来？
- 输出是自由文本，还是需要固定格式？
- 如果是分类任务，类别含义是否清楚？
- 如果模型答错，是缺少示例，还是任务说明本身含糊？

**零样本提示最适合作为第一步：先用它快速获得一个基线结果，再根据错误类型决定是补充上下文、指定输出格式、加入示例，还是升级到更复杂的提示技术。**
